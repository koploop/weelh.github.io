一次更换Vexo主题的经历

2017-07-02

最近工作不忙,所以打算更新下Blog,以前的主题现在有点看不顺眼,就在官网上找了一个很中意的主题Vexo(Vue.js官网风格). 本以为thems文件夹下add vexo就行了,谁知道这一折腾就是半晌…

Tags 和 Archives 目录不显示

作者描述要通过命令 cp themes/vexo/_source/* source/ 拷贝模板,但是这条命令在我的Mac上是没有效果的,命令行有一个提示,最初的我没有注意,所以走了很多弯路…
这里说明下,需要在 cp 后加上 -R 代表复制目录下的所有文件.

Project目录不显示

这个是因为需要在 _config.yml 中增加一条 :github_username: your_github_username 而这个关键信息作者在文档中并没有说明,是我给作者发了邮件后得到的回复.

error deployer not found:git

报这个错的时候请尝试使用下面的命令
npm install hexo-deployer-git --save

是否需要安装Hexo渲染器

这个我实在没有精力验证了,因为我一开始没有注意到命令行中 cp themes/vexo/_source/* source/ 复制无效的提示,所以在发现Tags目录不显示之后,就去尝试了其他办法,其中一位博主说要安装渲染器, 文章在这里. 在安装渲染器的过程中,报错无数,这期间我升级了 node.js , 重装了 npm ,替换了 npm 的源…终于渲染器成功了.

针对上个问题的说明

安装渲染器还有一个原因是因为我在 hexo g 生成静态文件的时候,有一个报错, 应该是Process layout.ejs 或者 Process archive.ejs 的函数报错,执行不成功,命令行提示的可能原因是 node.js 不是最新版本,或者 npm 没有安装,所以我看了下自己的版本,确实有点低, 这里我就顺便升级了下 node 和 npm , 再加上安装了渲染器,后面就成功了, 所以有类似问题的同学可以先尝试确认下自己的版本,再决定是否安装渲染器.

以上就是我再更换为Vexo主题上走过的坑,希望可以帮到同样用这个主题的你.
最后还是要非常感谢作者yanm1ng 给我们提供了这么漂亮的一个主题,以及回复我的邮件,感谢!
