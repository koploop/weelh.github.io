修改Git Commit Date

2017-10-05

某些情况下,你可能需要修改你的Git Commit Date来掩盖自己的偷懒,或者给你的Boss看…当然,通常不建议这么做.

修改本次提交时间

如果要恢复正常时间,关闭你的 Terminal 即可.

或者,我们可以来的专业一点:

修改某次提交时间

可是这时候你发现你已经提交了,再想修改时间怎么办呢?

如果你只是本地 commit 了,并没有 push 到 origin, 那这样就已经OK, 输入 git log 你会发现提交已经变成了你更改的时间了. 但是如果已经 push 了呢? 也没关系,紧接着执行下面两条命令即可:

Addition

author date 与 committer date 分别代表什么意思?

The author is the person who originally wrote the work, whereas the committer is the person who last applied the work

简单来说, author date代表的是作者执行 git commit 的时间, 而 committer date 代表的是这个提交被改变的时间.通常情况下和 author date 一样.

我们举例来说:

有两个分支 branch_A 和 branch_B , A 作为当前工作目录,我们增加了一个提交 a , 这时候我们要将两个分支进行 rebase 以合并功能. 那么合并之后. 之前提交的 a 的 committer date 已经发生了变化, 而 author date 不会变化.
